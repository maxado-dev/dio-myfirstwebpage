Button

Formulários

Para fazer um formulário, uso a tag form, preciso também especificar se ele vai ser do tipo get ou do tipo post, um para requisitar dados de um servidor, os dados são anexados na URL, ideal para buscas e filtros. O outro para enviar dados para um servidor, os dados vão no corpo de requisição, não na URL.

Dentro do form, vão os campos com as informações que quero trabalhar, além de no final pelo menos um button para enviar o formulário.

Formatando o texto

Temos diversas tags para formatar texto, vamos focar nas principais.

  • Strong

    Parecido com o negrito, porém com carga semântica, ressaltando grande importância.

  • Itálico

    Que é majoritariamente utilizado para citações e metáforas, mas o uso é diverso.

  • Negrito

    Clássico utilizado para destaque visual no texto, mas sem carga de importância atrelada.

  • Sublinhado

    Também utilizado para gerar destaque.

  • Sublinhado + Negrito

    Também é possível combinar ambos.

  • Mark

    Efeito de marca texto.

  • Sup

    Alinhamento superior, útil para potenciação.

  • Sub

    Alinhamento inferior.

  • Blockquote

    Alinha o texto mais à direita, utilizado para citações

Atributos de Tags

Tags também possuem atributos, são os mais diversos e dependem totalmente da tag, porém alguns são mais abrangentes:

  • id
  • style
  • class

E existem as específicas de tag também, como por exemplo:

Input

  • type="number"
  • type="text"
  • type="color"

Img

  • src="path/URL"
  • width="150"

Div e Span

Div e span são separadores que podem diferenciar os conteúdos, facilitando assim sua formatação pelo CSS.

Div

Isola um envolto de tags selecionado, tornando-às parte de si, posteriormente ao fazer referência à div, estou associando todas as tags dentro dela.

Span

Isola fragmentos menores, como o interior de uma caixa de texto ou botão que seja, para uma personalização ou ação mais precisa.

Fieldsets

Fieldsets são separadores de conteúdo, especialmente usado em formulários, para delimitar seus campos. É uma separação tanto no código quanto visual.

Embeds

Por meio da tag embed consigo inserir conteúdos diversos na página, podendo ser praticamente qualquer coisa.

Iframes

A tag Iframes veio para substituir os Embeds, sendo possível eu anexar qualquer tipo de conteúdo na página, com muito mais controle sobre o conteúdo e semântica também.

Cores

Embora o HTML seja capaz de atribuir cores por atributo, esta é a função do CSS hoje, e é visto como uma má prática.

Por css, podemos atribuir cor de fonte, de texto, de borda, e muito mais. Também pode-se escolher cor das seguintes formas:

  • RGB
  • HEX
  • HSL
  • RGBA
  • HSLA

Mídias

voltar

Além do Iframe ter a possibilidade de incluir media externa, temos outras tags para fins mais específicos e adequados.

Image

É possível adicionar imagens de diversos formatos, sendo os mais comuns:

  • jpeg - para imagens leves sem compromisso com a qualidade
  • png - para mais riqueza em detalhes com custo maior
  • svg - imagens vetoriais

Mesmo podendo inserir um SVG pela tag img, é recomendado utilizar a tag svg, por questões de semântica e uso.

Também existem algumas ferramentas de acessibilidade que são:

Alt

É uma boa prática adicionar o atributo alt nas imagens, com uma descrição bem detalhada para dar o contexto para ferramentas de acessibilidade.

Figcaption

Figcaption também pode colaborar para uma experiência mais agradável de usuário, com uma descrição breve da imagem.

Audio

Executa um áudio na página, sendo controlável pela tag controls, podendo ficar em loop ou dar autoplay, etc.

Video

Insere um video de acordo com a source indicada. Mais indicado que o Iframe quando não for algo explicitamente pertencente a outra fonte, como um vídeo do YouTube do canal de um terceiro por exemplo.

É capaz de controlar tamanho, largura, dentre diversas outras propriedades, além de poder utilizar formatos de vídeo secundário caso o usuário não consiga reproduzir o principal.

Track

Track consegue atribuir legenda e até mesmo capítulos ao vídeo, deixando o conteúdo mais claro e acessível.

Tabelas

voltar

Com a tag table consigo definir uma tabela. É possível personalizá-la com propriedades, porém o CSS faz isso melhor. O que podemos fazer aqui é adicionar um summary para ferramentas de acessibilidade, com uma ampla descrição dos dados.

Com tr, indico uma table row, sendo a primeira geralmente a th, ou table header.

Com uma nova tr, posso começar a inserir as linhas ou rows dentro de tds, ou table cell.

Para cada row, preciso de uma tr, para mover entre células, usa-se os td.

Semântica

voltar

A semântica de uma página diz respeito ao quanto ela por meio de suas tags estão num contexto que faz sentido, como o conteúdo por si só se conversa com as tags mais apropriadas e específicas possíveis.

Uma boa semântica é essencial para Search Engine Optimization e também para melhor desempenho de ferramentas de acessibilidade, ou seja, é um foco de atenção indispensável nos dias de hoje.

As principais tags para uma semântica estrutural básica são:

  • Header

    Cabeçalho, geralmente com informações da página.

  • Main

    O conteúdo principal.

  • Nav

    A parte de navegação.

  • Aside

    Espaço reservado para propagandas, ou coisas irrelevantes para o conteúdo em si.

  • Section

    Separador de conteúdo parecido com o div, porém utilizado quando o conteúdo em si tem relação entre eles, div mais genérico útil principalmente para fins estéticos, section para fins semânticos, mas também podendo ser utilizado para fins estéticos.

  • Footer

    Rodapé da página, com demais informações gerais.
